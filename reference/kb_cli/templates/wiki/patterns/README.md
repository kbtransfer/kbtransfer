# patterns/

Reusable problem-solution descriptions. Reference-style documentation
that other pages (and eventually, distilled packs) can cite.

A good pattern page answers, in order:

1. **Problem.** What recurring situation does this address?
2. **Context.** When does the problem appear? What is out of scope?
3. **Solution.** The approach itself, with enough detail to reproduce.
4. **Consequences.** What the solution costs; known failure modes.
5. **Related.** Links to adjacent patterns, decisions, and entities.

Patterns are the unit that distills cleanly into an AutoEvolve pack.
If a page grows beyond ~5 pages of content, split it.
